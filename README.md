# 🦈 Shark Attack Data Wrangling & Analysis

## 📌 Overview
This project focuses on cleaning and analyzing a historical shark attack dataset using Python and pandas. The goal is to transform raw, messy data into meaningful insights to support decision-making in a tourism and water activity business context.

## 🎯 Objectives
We investigate the following key questions:

- Which resort regions have lower incident rates?  
- Are shark attacks concentrated during certain months?  
- Should some water activities be promoted more than others?  

## 📊 Dataset
- Source: Global Shark Attack File (GSAF)  
- Format: Excel (.xls)  
- Key variables: Date, Country, Activity  

## ⚙️ Tools Used
- Python (Pandas, NumPy)  
- Jupyter Notebook / Google Colab  
- Matplotlib / Seaborn  

## 🧹 Data Cleaning
We applied multiple techniques to prepare the dataset:
- Handling missing values  
- Removing duplicates  
- Standardizing text (countries, activities)  
- Formatting dates  
- Filtering irrelevant records  

## 📈 Key Insights

### 🌍 Regions
- Higher incidents in USA and Australia  
- Lower rates in less-populated or colder regions  
Safer destinations can be recommended accordingly

### 📅 Seasonality
- Attacks peak in summer months (July–September)  
Risk increases with human water activity

### 🏄 Activities
- Higher risk: Surfing, swimming  
- Lower risk: Boating, controlled fishing  
Promote lower-risk activities in high-risk areas

## 🧪 Workflow
1. Data exploration & hypothesis definition  
2. Data cleaning & transformation  
3. Exploratory Data Analysis (EDA)  
4. Insights & presentation

## 📌 Conclusion
Shark attack risk is influenced by location, seasonality, and activity type. These insights can help businesses recommend safer destinations and activities.
